# Investigate override duration semantics and backspace encoding

Discovered during the CRT-I-0029 Test Coverage Sprint. Two potential issues to investigate.

## Issue 1: OverrideState duration_ms=0 semantics contradiction

**Location**: `src/window.rs` — `ActiveOverride::is_active()` (line ~281) and `OverrideState` comment

**Problem**: The `EventOverride` doc comment says `duration_ms: 0` means "persist until cleared by another event" (used for `::on-blur`). However, `ActiveOverride::is_active()` checks `self.triggered_at.elapsed() < duration` — when duration is 0ms, `elapsed() >= 0` is always true, so the override expires immediately.

**Current impact**: Low. The `on_blur` use case (the only intended persistent override) appears to use a non-zero duration in practice, so this code path isn't hit. But if someone configures `duration_ms: 0` expecting "persist forever", it would silently expire.

**Expected vs Actual**:
- Expected: `duration_ms: 0` overrides persist until explicitly cleared
- Actual: `duration_ms: 0` overrides expire immediately

**Fix options**:
1. Add a special case in `is_active()`: `if self.properties.duration_ms == 0 { return true; }`
2. Update the doc comment to say 0 means "expire immediately" and use a large value for persist
3. Add a separate `persistent: bool` field to `EventOverride`

## Issue 2: Backspace encoding produces DEL not BS

**Location**: `src/input/key_encoder.rs` — termwiz KeyCode::Backspace encoding

**Problem**: termwiz encodes Backspace as DEL (0x7F) rather than BS (0x08). This is actually the correct modern terminal behavior (most terminals send DEL for Backspace), but it's worth documenting since it could surprise someone expecting BS.

**Current impact**: None — this is correct behavior. Just worth a doc comment in the code.

**Action**: Add a comment in `key_encoder.rs` noting that Backspace intentionally produces DEL (0x7F) per modern terminal convention (VT220+), not BS (0x08).
