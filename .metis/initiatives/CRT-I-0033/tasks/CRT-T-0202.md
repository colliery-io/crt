## Objective **[REQUIRED]**

Implement the pure, I/O-free core of file-path detection in `src/input/mod.rs`, mirroring
the existing URL detection functions: a `DetectedPath` type, a per-line tokenizer for path
candidates, a trailing `:line[:col]` suffix parser, trailing-punctuation trimming, and
position-lookup helpers. No filesystem access in this task — resolution and existence
validation are CRT-T-0203.

## Acceptance Criteria **[REQUIRED]**

- [ ] `DetectedPath` struct exists (mirrors `DetectedUrl`): raw token span (`start_col`,
  `end_col`, `line`, `end_line`), the path string with any suffix stripped, and optional
  `line`/`col` from the suffix.
- [ ] `detect_paths_in_line(line_text, line_num)` returns candidates for tokens beginning
  with `/`, `~/`, `./`, `../`, or a bare multi-segment token containing ≥1 `/`.
- [ ] Single-segment bare words (no `/`, e.g. `README`) are NOT detected.
- [ ] Trailing prose punctuation is trimmed (share/reuse `trim_url_trailing_punctuation`).
- [ ] A trailing `:N` or `:N:M` suffix is parsed into `line`/`col` and removed from the
  path string; a `:` not followed by a trailing number (e.g. inside a name) is not misread.
- [ ] `find_path_at_position`, `find_path_index_at_position`, `is_position_in_path` exist
  and mirror the URL position helpers.
- [ ] Unit tests cover each path form, suffix variants (`:42`, `:42:7`, none, colon-in-name),
  punctuation trimming, and position hit/miss.
- [ ] No filesystem access introduced in this task.

## Implementation Notes

### Technical Approach
Mirror the URL code in `src/input/mod.rs` (`DetectedUrl` L28–41, `detect_urls_in_line`
L97–115, position helpers L118–145, `trim_url_trailing_punctuation` L76–94). Reuse the
punctuation-trim logic rather than duplicating it. Keep everything pure so it is unit
testable without a terminal or disk.

### Dependencies
None — foundational task. Unblocks CRT-T-0203, CRT-T-0204, CRT-T-0205.

### Risk Considerations
Over-broad bare-relative grammar causes false positives; existence validation in
CRT-T-0203 is the backstop, but keep the grammar conservative (require ≥1 `/`).

## Status Updates **[REQUIRED]**

**Completed** (2026-06-30).

Implemented in `src/input/mod.rs`:
- `DetectedPath` struct (mirrors `DetectedUrl`): `path` (suffix-stripped), `target_line`,
  `target_col`, `exists` (defaults `false`, set by CRT-T-0203), and the `start_col`/
  `end_col`/`line`/`end_line` span (span covers the whole token *including* the suffix).
- `path_regex()` — maximal path-character run; tokens filtered in code.
- `detect_paths_in_line(line_text, line_num)` — skips URLs (`://`), trims trailing prose
  punctuation (reuses `trim_url_trailing_punctuation`), parses the suffix, and keeps only
  path-like tokens (`is_path_like` requires ≥1 `/`). No I/O.
- `parse_path_suffix` — splits trailing `:N` / `:N:M`; non-numeric trailing `:word` stays
  in the path.
- `find_path_at_position` / `find_path_index_at_position` / `is_position_in_path`.
- Refactor: extracted `is_position_in_span(...)` shared by URL and path hit-testing;
  `is_position_in_url` now delegates to it (no behavior change).

Tests: 13 new unit tests (each path form, URL exclusion, suffix variants incl.
colon-in-name, punctuation trim, span-includes-suffix, multiple paths, position hit/miss).
`cargo test --bin crt` input module: 28 passed, 0 failed (URL tests still green).
