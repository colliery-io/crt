## Status Updates **[REQUIRED]**

**Completed** (2026-06-30).

**Edge cases** (tests in `src/input/mod.rs`, decisions documented):
- Directory targets → clickable (`validate_directory_is_clickable`); open via OS default.
- `:line` on a missing file → not clickable (`validate_line_suffix_on_missing_file_not_clickable`).
- Symlinks → followed by `try_exists` (`validate_follows_valid_symlink`, unix-gated); broken
  symlinks report non-existent (acceptable).
- Paths with spaces → out of scope for detection (the tokenizer splits on whitespace);
  documented in README. The *open* path round-trips spaces correctly (builder test).
- Multi-line wrapped paths → out of scope (paths wrap rarely; URL-style merge not applied).

**NFR-001 perf**: `validate_caches_full_screen_of_duplicates` (200 identical tokens → exactly
1 filesystem check) and `validate_respects_per_pass_budget` (>`MAX_STAT_PER_PASS` distinct
tokens → checks capped at 256, overflow treated as non-existent). Cache + budget verified.

**Cross-platform**: verified by construction — modifier handling reuses the existing
`#[cfg(target_os)]` super/control split (macOS Cmd / Linux Ctrl), and CWD comes from the
pre-existing `get_process_cwd` (lsof on macOS, `/proc/<pid>/cwd` on Linux). No new
platform-specific code paths added (the only `#[cfg]` is the unix symlink test).

**Docs**: README updated — Cmd+Click reference now says "URL or file path", and a new
"Opening files" subsection documents path forms, existence/CWD resolution, the
`open_file_command` option with `{file}`/`{line}`/`{col}` examples (VS Code, Vim), and the
CWD/scrollback limitation. Starter config template also carries a commented example.

**Tests**: full `cargo test` green — bin 238 passed; workspace crates 22/46/19 passed; 0
failed. `cargo clippy --bin crt` introduces no warnings in the new code (remaining warnings
are all pre-existing). Visual/golden tests untouched (macOS-only constraint respected).
