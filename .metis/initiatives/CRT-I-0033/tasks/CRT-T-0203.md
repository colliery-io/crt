## Objective **[REQUIRED]**

Turn detected path tokens into validated, clickable targets: resolve `~`/relative/absolute
forms against `$HOME` and the shell CWD, gate clickability on on-disk existence, and cache
results so per-frame detection does not trigger a `stat` storm (NFR-001).

## Acceptance Criteria **[REQUIRED]**

- [ ] `resolve_path(token, cwd, home) -> PathBuf`: `~`/`~/...` → `$HOME`; relative →
  joined onto `cwd`; absolute used as-is.
- [ ] Existence gate via `Path::try_exists()` — only existing paths are marked clickable;
  the result is carried on `DetectedPath` (e.g. an `exists`/validated flag).
- [ ] A `(cwd, token) -> bool` cache avoids re-stat'ing the same token within a frame and
  across frames where the rendered line set is unchanged; cache invalidates when `cwd`
  changes.
- [ ] The number of `stat`/`try_exists` calls per frame is bounded (cap + `log` when the
  cap is hit) so detection cannot stall the render loop.
- [ ] Unit tests using a temp dir cover: existing vs missing file, `~` expansion, relative
  resolution against a supplied cwd, absolute passthrough, and cache invalidation on cwd
  change.

## Implementation Notes

### Technical Approach
Pure resolution/validation functions that take `cwd` and `home` as parameters (do not read
globals) so they are testable with temp dirs. The cache lives with the per-frame detection
state in the interaction layer (wired in CRT-T-0205); design its API here. CWD source is
`active_shell_cwd()` (`src/window/mod.rs` ~L529) → `crates/crt-core` `working_directory()`.

### Dependencies
Depends on CRT-T-0202 (`DetectedPath`). Unblocks CRT-T-0205.

### Risk Considerations
`try_exists` on a slow/network FS could add latency — the per-frame cap and caching are the
mitigations. Symlink/permission cases: treat "exists" liberally (a broken symlink may
report not-exists; acceptable). CWD reflects the *current* foreground process, not the dir
active when scrollback was produced (documented limitation in the initiative).

## Status Updates **[REQUIRED]**

**Completed** (2026-06-30).

Implemented in `src/input/mod.rs`:
- `resolve_path(token, cwd, home) -> Option<PathBuf>` — pure: `~`/`~/...` → home, absolute
  as-is, relative joined onto cwd; returns `None` when context is missing or for unsupported
  `~user` forms.
- `PathValidator` — holds the `(resolved path) -> bool` existence cache plus cwd/home
  context. `begin_pass(cwd, home)` clears the cache when cwd changes and resets the per-pass
  budget; `validate(&mut DetectedPath)` sets `exists` via `try_exists`, using the cache and a
  `MAX_STAT_PER_PASS = 256` budget (logs a warning once when exceeded). `validate_all` for a
  slice; `stats_this_pass()` accessor for observability/tests.

Design notes:
- Cache keyed by the *resolved absolute path* so absolute/home tokens stay cached across cwd
  changes while relative tokens naturally re-key; explicit clear on cwd change satisfies the
  invalidation requirement.
- Budget bounds per-frame `stat`s (NFR-001); cross-frame caching avoids re-stat on unchanged
  content.

Tests: 7 new unit tests (resolve absolute/home/relative incl. missing-context and `~user`;
validate existing vs missing via tempdir; `~` expansion; cache prevents re-stat
(`stats_this_pass == 1`); cache invalidation on cwd change). All pass.

Note: `DetectedPath`/detection/validator currently warn as dead_code in the binary build —
expected; they get wired into the app in CRT-T-0205, which clears the warnings.
