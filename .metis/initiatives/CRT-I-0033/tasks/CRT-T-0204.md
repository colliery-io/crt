## Objective **[REQUIRED]**

Add the `open_file_command` config option and an `open_file` action that opens a path via
the OS default (`open::that`) by default, or via the configured editor command (with
`{file}`/`{line}`/`{col}` substitution) when one is set. Mirrors `open_url`.

## Acceptance Criteria **[REQUIRED]**

- [ ] `config.rs` gains an optional `open_file_command: Option<String>` (placed consistently
  with existing config, e.g. alongside `shell.working_directory`), parsed from the config
  file and defaulting to `None`.
- [ ] A pure builder turns `(command_template, file, line, col)` into the argv/command to
  run, substituting `{file}`, `{line}`, `{col}`; missing `line`/`col` use sensible defaults
  (e.g. line `1`, col `1`) or are omitted per template.
- [ ] `open_file(path, line, col, cfg)`: if `open_file_command` is `Some`, spawn the built
  command; otherwise `open::that(path)` (line/col ignored for OS-default).
- [ ] Errors are logged like `open_url` (`log::error!`), never panic.
- [ ] Unit tests cover the substitution builder for: command with `{file}` only, with
  `{file}:{line}`, with `{file}:{line}:{col}`, and a path containing spaces — without
  actually spawning a process.

## Implementation Notes

### Technical Approach
Model on `open_url` (`src/input/mod.rs` ~L214–225). Keep the placeholder-substitution logic
a pure function returning the command + args so it is unit-testable; the spawn is a thin
wrapper. Decide argv splitting strategy (shell-word split vs. fixed `program arg...`) and
document it. Config wiring mirrors existing fields in `src/config.rs`.

### Dependencies
Loosely depends on CRT-T-0202 for the `line`/`col` carried on `DetectedPath`; can be built
in parallel with CRT-T-0203. Unblocks CRT-T-0205.

### Risk Considerations
Command injection / quoting: substitute into discrete argv elements rather than a shell
string where possible; if a shell string is used, document the trust assumption (user's own
config). Spaces in paths must round-trip correctly.

## Status Updates **[REQUIRED]**

**Completed** (2026-06-30).

- `src/config.rs`: added `Config.open_file_command: Option<String>` (top-level, defaults to
  `None` via `#[serde(default)]`), with a doc comment and a commented example in the starter
  `DEFAULT_CONFIG_TEMPLATE` (`code -g {file}:{line}:{col}`).
- `src/input/mod.rs`:
  - `build_open_command(template, file, line, col) -> Option<(String, Vec<String>)>` — pure:
    splits template on whitespace, substitutes `{file}`/`{line}`/`{col}` (missing line/col
    default to `1`), returns `None` for empty templates. Splitting before substitution keeps
    a `{file}` token a single argv element even with spaces in the path.
  - `open_file(path, line, col, command_template)` — spawns the built command when a template
    is set, else falls back to `open::that(path)` (line/col ignored). Errors logged like
    `open_url`, never panics.

Decision: argv built as `program + Vec<args>` (no shell), so paths with spaces round-trip
safely and there's no shell-injection surface; trust assumption is the user's own config.

Tests: 6 new builder unit tests ({file} only, +line, +line:col, default line/col, path with
spaces stays one arg, empty template → None). All pass.
