## Implementation Notes

### Technical Approach
Mirror the URL plumbing: `detected_urls`/`hovered_url_index` in `interaction.rs`, the
detection block in `window/mod.rs`, `update_hover` and the Cmd+click handler in `mouse.rs`
(~L235â€“246, `MouseClickTarget::OpenUrl`). Reuse the existing modifier detection
(`super_key()` / `control_key()`). Keep the click-target resolution pure for testing.

### Dependencies
Depends on CRT-T-0202, CRT-T-0203, CRT-T-0204. Unblocks CRT-T-0206.

### Risk Considerations
Hover precedence between URL and path must be unambiguous (URL wins) to avoid flicker or
double-underline. Ensure the per-frame population reuses the cache so wiring doesn't
reintroduce a stat storm.

## Status Updates **[REQUIRED]**

**Completed** (2026-06-30).

- `src/window/interaction.rs`: `InteractionState` gains `detected_paths: Vec<DetectedPath>`,
  `hovered_path_index: Option<usize>`, and `path_validator: PathValidator` (all `Default`).
- `src/window/mod.rs`: after URL detection, runs `detect_paths_in_line` over the cached line
  texts, then `path_validator.begin_pass(active_shell_cwd(), $HOME)` + `validate_all`, and
  `retain(|p| p.exists)` so only existing paths remain. Reuses the cross-frame cache (NFR-001).
- `src/input/mouse.rs`:
  - `handle_cursor_moved` now updates `hovered_path_index`; URLs take precedence (a path
    hover is only considered when no URL is hovered) so overlaps never double-underline.
  - Added pure `resolve_click_link(urls, paths, col, line) -> ClickLink` (URL precedence) and
    a `MouseClickTarget::OpenFile` variant.
  - `handle_mouse_input` gained an `open_file_command: Option<&str>` param; on Cmd/Ctrl+click
    it matches `resolve_click_link`, opening the URL or re-resolving the path token against
    cwd/$HOME and calling `open_file` with the parsed line/col + configured command.
- `src/app/handler.rs`: call site passes `self.config.open_file_command.as_deref()`.

Modifier handling reuses the existing `#[cfg(target_os)]` super/control split (Linux
Ctrl+click parity).

Tests: 4 new `resolve_click_link` tests (picks url, picks path, URL precedence on overlap,
none when empty). `cargo build --bin crt` green; full input module suite 135 passed, 0 failed.

Note: `MouseClickTarget`/`determine_click_target`/`compute_click_count` remain dead_code â€”
that is pre-existing (the live dispatch uses `resolve_click_link`/inline checks, not the enum).
