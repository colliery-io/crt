# Extract PTY update processing from render_frame()

**Phase 1 of render pipeline decomposition. No dependencies — can start immediately.**

## Parent Initiative

[[CRT-I-0028]]

## Objective

Extract the PTY output processing logic from `render_frame()` (`src/render/mod.rs:82-98`) into a standalone function that can be called and tested independently of GPU rendering.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] New function `process_pty_updates()` exists in `src/render/mod.rs` (or a new sub-module)
- [ ] Function takes the active shell and returns a struct indicating: whether content changed, shell events collected, and any title change
- [ ] `render_frame()` calls this new function instead of inline PTY processing
- [ ] All existing tests pass unchanged (`cargo test --all-targets`)
- [ ] No functional behavior change — purely a code move

## Implementation Notes

### Target Code

`src/render/mod.rs:82-98` — the block that:
1. Gets active tab ID from tab bar
2. Calls `shell.process_pty_output()` and sets `dirty` flag
3. Invalidates content hash on change
4. Calls `shell.take_shell_events()` for bell/command events
5. Updates tab title on title change

### Proposed Signature

```rust
pub struct PtyUpdateResult {
    pub content_changed: bool,
    pub shell_events: Vec<ShellEvent>,
    pub title_change: Option<String>,
}

pub fn process_pty_updates(shell: &mut ShellTerminal) -> PtyUpdateResult
```

The caller (`render_frame`) applies the result to state:
```rust
let result = process_pty_updates(shell);
if result.content_changed {
    state.render.dirty = true;
    state.content_hashes.insert(tab_id, 0);
}
if let Some(title) = result.title_change {
    state.gpu.tab_bar.set_tab_title(tab_id, title);
}
```

### Risk Considerations

Low risk — this is a straightforward extraction with no logic change. The active tab lookup and state mutation remain in `render_frame()`.
