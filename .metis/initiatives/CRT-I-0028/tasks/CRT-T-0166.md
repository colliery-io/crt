# Make ShellTerminal generic over PtyBackend and add MockPty

**Phase 2 — PTY abstraction. Depends on CRT-T-0165 (PtyBackend trait).**

## Parent Initiative

[[CRT-I-0028]]

## Objective

Make `ShellTerminal` generic over the `PtyBackend` trait so tests can inject a `MockPty` instead of spawning real shell processes. Create the `MockPty` implementation that supports scripted output sequences and input capture.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `ShellTerminal<P: PtyBackend>` is generic over its PTY backend
- [ ] Type alias `pub type ShellTerminal = ShellTerminalGeneric<Pty>` preserves backward compatibility
- [ ] `MockPty` struct exists (in `#[cfg(test)]` or as a public test utility)
- [ ] `MockPty` supports: pre-loading output bytes, capturing written input, fake resize tracking
- [ ] A smoke test demonstrates creating `ShellTerminalGeneric<MockPty>` and processing scripted output
- [ ] All existing tests (including `tests/shell_tests.rs`) pass unchanged
- [ ] All call sites that use `ShellTerminal` continue to work without modification

## Implementation Notes

### ShellTerminal Generification

```rust
pub struct ShellTerminalGeneric<P: PtyBackend> {
    terminal: Terminal,
    pty: P,
}

// Backward-compatible alias
pub type ShellTerminal = ShellTerminalGeneric<Pty>;
```

All existing methods on `ShellTerminal` that call `self.pty.write()`, `self.pty.try_read()`, etc. work unchanged because `Pty` implements `PtyBackend`.

### MockPty Design

```rust
#[cfg(test)]
pub struct MockPty {
    /// Pre-loaded output chunks (consumed in order by try_read/read_available)
    output_queue: VecDeque<Vec<u8>>,
    /// Captured input (everything written via write())
    captured_input: Vec<u8>,
    /// Last resize dimensions
    last_resize: Option<(u16, u16)>,
    /// Whether shutdown was called
    shutdown_called: bool,
}

impl MockPty {
    pub fn new() -> Self { /* empty queues */ }
    pub fn with_output(chunks: Vec<Vec<u8>>) -> Self { /* pre-load */ }
    pub fn captured_input(&self) -> &[u8] { &self.captured_input }
}
```

### Smoke Test

```rust
#[test]
fn test_mock_pty_terminal() {
    let mock = MockPty::with_output(vec![b"Hello, world!\r\n".to_vec()]);
    let mut term = ShellTerminalGeneric::with_mock(Size::new(80, 24), mock);
    assert!(term.process_pty_output());  // Processes pre-loaded output
    let text = term.terminal().all_lines_text();
    assert!(text.iter().any(|(_, line)| line.contains("Hello, world!")));
}
```

### Risk Considerations

Medium risk — `ShellTerminal` is used throughout the codebase (main.rs, window.rs, render/mod.rs). The type alias should make this transparent, but all HashMap types like `HashMap<TabId, ShellTerminal>` need to resolve correctly. May need to verify with `cargo check` that all call sites work.

## Status Updates

- Renamed `ShellTerminal` struct to `ShellTerminalGeneric<P: PtyBackend>` with type alias `ShellTerminal = ShellTerminalGeneric<Pty>`
- Pty-specific constructors (new, with_cwd, with_shell, with_options, pty()) stay on `impl ShellTerminal`
- Generic methods (process_pty_output, send_input, resize, terminal, etc.) moved to `impl<P: PtyBackend> ShellTerminalGeneric<P>`
- Added `with_backend(size, pty)` constructor for creating with any PtyBackend
- Added `MockPty` in test module with output queue, input capture, resize tracking
- Added `test_mock_pty_terminal` smoke test verifying terminal content, input capture, and resize
- All existing tests pass unchanged — type alias provides full backward compatibility
