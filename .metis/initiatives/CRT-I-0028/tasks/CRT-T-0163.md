# Extract effect patch computation from render_frame()

**Phase 1 of render pipeline decomposition. Independent of T-0161/T-0162.**

## Parent Initiative

[[CRT-I-0028]]

## Objective

Extract the effect patch decision logic from `render_frame()` (`src/render/mod.rs:156-229`) into a pure function. This ~70-line block determines which visual effects (starfield, particles, matrix, rain, grid, shape, sprite) need their configuration patched based on the current override state. Currently, the decision logic is interleaved with `effects_renderer` mutations.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] New pure function `compute_effect_patches()` takes override state and returns a list of (effect_name, EffectConfig) patches to apply
- [ ] Function does not call `effects_renderer.apply_effect_patch()` directly — it only returns the data
- [ ] Function does not call `state.ui.overrides.set_patched()` — caller handles that
- [ ] `render_frame()` calls this function, then applies patches to the effects renderer
- [ ] The `is_patched()` / `set_patched()` tracking continues to work correctly
- [ ] All existing tests pass unchanged
- [ ] No functional behavior change

## Implementation Notes

### Target Code

`src/render/mod.rs:156-229` — the block that for each effect type:
1. Checks if override has a patch for that effect (`get_starfield_patch()`, etc.)
2. Checks if already patched (`is_patched("starfield")`)
3. If not patched, converts to EffectConfig and calls `effects_renderer.apply_effect_patch()`
4. Marks as patched via `set_patched()`

### Proposed Signature

```rust
pub struct EffectPatchRequest {
    pub effect_name: String,  // or EffectId enum if CRT-I-0030 lands first
    pub config: EffectConfig,
}

pub fn compute_effect_patches(overrides: &OverrideState) -> Vec<EffectPatchRequest>
```

### Complexity Note

This is the most complex extraction in Phase 1 because the current code checks `is_patched()` state (which is mutable) to avoid re-applying patches. The pure function needs to return only _new_ patches. The caller then applies them and updates the patched set. Careful attention to the `clear_patches()` path (when overrides expire) is needed.
