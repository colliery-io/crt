## Implementation Notes

### Duplicated Code

Both functions contain identical blocks for:
1. Creating mpsc channels for input/output (`pty.rs:~140-145` and `~248-252`)
2. Spawning reader thread (`pty.rs:147-164` and `254-271`) — reads from PTY master, sends Vec<u8> to channel
3. Spawning writer thread (`pty.rs:167-200` and `274-307`) — receives PtyInput from channel, writes to PTY
4. Constructing the Pty struct with channels and child handle

### Proposed Helper

```rust
fn spawn_pty_threads(
    pair: portable_pty::PtyPair,
    child: Box<dyn Child + Send + Sync>,
) -> Pty {
    let (input_tx, input_rx) = mpsc::channel::<PtyInput>();
    let (output_tx, output_rx) = mpsc::channel::<Vec<u8>>();

    // Reader thread
    let mut reader = pair.master.try_clone_reader().expect("...");
    thread::spawn(move || { /* read loop */ });

    // Writer thread
    let mut writer = pair.master.take_writer().expect("...");
    thread::spawn(move || { /* write loop */ });

    Pty { input_tx, output_rx, child }
}
```

Both `spawn_with_options()` and `spawn_with_cwd()` handle their specific setup (environment variables, shell detection, init scripts) and then call `spawn_pty_threads(pair, child)`.

### Risk Considerations

Low risk — the thread logic is identical between the two functions. The only difference is in the setup code before thread spawning. Careful to preserve error handling in reader/writer threads.
