# Split mouse handler into pure decision + side-effectful application

**Phase 3 — Input handler refactoring. Should follow CRT-T-0168 (reuses InputContext pattern).**

## Parent Initiative

[[CRT-I-0028]]

## Objective

Apply the same pure/applied split pattern from CRT-T-0168 to the mouse input handlers in `src/input/mouse.rs`. Extract pure hit-testing and action determination from state-mutating mouse handlers.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] Mouse hit-testing logic (which UI element was clicked: tab bar, terminal area, context menu, URL) is extractable as a pure function given coordinates and layout dimensions
- [ ] `MouseAction` enum defined for mouse decisions (similar to KeyboardAction)
- [ ] Pure functions for: coordinate-to-grid-position conversion, URL hover detection, tab bar hit testing
- [ ] Side-effectful functions for: selection updates, context menu state changes, clipboard operations
- [ ] `handle_cursor_moved()`, `handle_mouse_input()`, `handle_mouse_wheel()` remain as composition wrappers
- [ ] At least 3 unit tests for pure hit-testing functions
- [ ] All existing tests pass unchanged

## Implementation Notes

### Current Structure

`src/input/mouse.rs` has three main entry points:
1. `handle_cursor_moved(state, x, y)` — updates hover state, selection drag, URL detection
2. `handle_mouse_input(state, button, element_state, modifiers)` — handles clicks
3. `handle_mouse_wheel(state, delta, modifiers)` — handles scrolling

All three directly access `state.gpu.tab_bar`, `state.gpu.glyph_cache`, `state.interaction`, `state.ui.context_menu`, etc.

### Key Pure Extractions

**1. Coordinate-to-grid conversion:**
```rust
pub fn screen_to_grid_position(
    x: f32, y: f32,
    content_offset: (f32, f32),
    padding: f32,
    cell_width: f32,
    line_height: f32,
) -> Option<(usize, usize)>  // (col, row)
```

**2. Click target determination:**
```rust
pub enum ClickTarget {
    TabBar { tab_index: usize },
    TabClose { tab_index: usize },
    Terminal { col: usize, row: usize },
    ContextMenu { item_index: usize },
    Url { url_index: usize },
    Background,
}

pub fn determine_click_target(
    x: f32, y: f32,
    tab_bounds: &[Rect],
    terminal_bounds: Rect,
    context_menu: Option<&ContextMenuBounds>,
    url_regions: &[UrlRegion],
) -> ClickTarget
```

**3. URL hover detection:**
Already partially pure in `find_url_at_position()` / `find_url_index_at_position()` — these just need the coordinate conversion extracted.

### Pragmatic Scope

Mouse handling involves more coordinate math and less branching than keyboard handling. Focus on extracting the hit-testing and coordinate conversion as pure functions. Selection tracking (drag state machine) can remain stateful for now — it's inherently sequential state.

### Dependencies

Reuses `InputContext` from CRT-T-0168 where applicable (e.g., checking if context menu is visible, if search is active).
