# Separate terminal render data preparation from GPU submission

**Phase 1 of render pipeline decomposition. Largest task — the core separation. Can be done after T-0161/T-0162/T-0163 for a cleaner diff, but not strictly dependent.**

## Parent Initiative

[[CRT-I-0028]]

## Objective

Separate the logic that collects terminal cell content, cursor info, selection ranges, and UI state from the code that submits it to GPU renderers. This is the bulk of `render_frame()` (~lines 300-800) and the highest-value extraction — it enables testing what _should_ be rendered via MockRenderer without any GPU.

## Implementation Notes

### Target Code

The middle section of `render_frame()` that:
1. Gets terminal renderable content
2. Computes content hash (skip if unchanged)
3. Iterates grid cells collecting CellContent structs
4. Collects cursor position and shape
5. Collects selection ranges
6. Applies theme colors (foreground/background overrides)
7. Collects text decorations (underlines for URLs, semantic zones)
8. Prepares tab bar render info

### Proposed Approach

Rather than one massive struct, use the existing trait types:

```rust
pub struct FrameRenderData {
    pub cells: Vec<CellContent>,
    pub cursor: Option<CursorInfo>,
    pub selections: Vec<SelectionRange>,
    pub tabs: Vec<TabRenderInfo>,
    pub tab_bounds: Rect,
    pub bell_flash: Option<f32>,
    pub search_matches: Vec<SearchHighlight>,
    pub decorations: Vec<TextDecoration>,
}

pub fn prepare_render_data(
    terminal: &Terminal,
    theme: &Theme,
    overrides: &OverrideState,
    // ... other read-only state
) -> Option<FrameRenderData>  // None if content hash unchanged
```

### Risk Considerations

This is the largest and most complex extraction. The current code accesses many fields of WindowState. The key challenge is identifying the minimal read-only inputs needed. May need to pass a reference struct rather than individual fields.

Recommend doing T-0161, T-0162, T-0163 first to reduce the remaining code in `render_frame()` before this extraction.
