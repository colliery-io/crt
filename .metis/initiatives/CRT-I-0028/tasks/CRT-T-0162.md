# Extract shell event override computation from render_frame()

**Phase 1 of render pipeline decomposition. Depends on CRT-T-0161 (uses its PtyUpdateResult).**

## Parent Initiative

[[CRT-I-0028]]

## Objective

Extract the shell event → theme override mapping logic from `render_frame()` (`src/render/mod.rs:99-145`) into a pure function. This logic determines which theme overrides to activate when shell events occur (bell → on_bell override, command success → on_command_success override, etc.).

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] New pure function `compute_shell_event_overrides()` takes shell events and theme config, returns override activations
- [ ] Function has no side effects — does not mutate OverrideState or any other state
- [ ] `render_frame()` calls this function and applies the returned overrides to state
- [ ] All existing tests pass unchanged
- [ ] No functional behavior change

## Implementation Notes

### Target Code

`src/render/mod.rs:99-145` — the block that:
1. Gets current theme from effect pipeline
2. Iterates shell events (Bell, CommandSuccess, CommandFail)
3. Maps each event to its theme override (theme.on_bell, theme.on_command_success, etc.)
4. Calls `state.ui.overrides.add()` with the override properties
5. Sets dirty flag and marks bell active

### Proposed Signature

```rust
pub struct OverrideActivation {
    pub event_type: OverrideEventType,
    pub properties: OverrideProperties,
    pub duration: Option<f32>,
}

pub fn compute_shell_event_overrides(
    shell_events: &[ShellEvent],
    theme: &Theme,
) -> Vec<OverrideActivation>
```

The caller applies each activation:
```rust
for activation in compute_shell_event_overrides(&result.shell_events, theme) {
    state.ui.overrides.add(activation.event_type, activation.properties, activation.duration);
    state.render.dirty = true;
}
```

### Key Benefit

This function is a pure data transformation — given events and a theme, compute overrides. It can be tested with any theme configuration and any combination of shell events without GPU state.
