# Split main.rs into app/ module directory

## Parent Initiative

[[CRT-I-0030]] — Readability and Module Decomposition

## Objective

Split `src/main.rs` (1,714 lines) into an entry point + `src/app/` module directory, separating the App struct, event handler, initialization, and menu action handling into focused files.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `src/main.rs` reduced to entry point only (fn main, event loop setup, minimal glue)
- [ ] `src/app/mod.rs` — App struct definition and field declarations
- [ ] `src/app/handler.rs` — `ApplicationHandler` impl (window_event, about_to_wait, etc.)
- [ ] `src/app/initialization.rs` — Theme loading, config setup, GPU initialization
- [ ] `src/app/menu_actions.rs` — `handle_menu_action()` and related helpers (~195 lines, 30+ match arms)
- [ ] All existing `use` paths and cross-references compile correctly
- [ ] Zero logic changes — pure file reorganization
- [ ] All 229 existing tests pass without modification

## Implementation Notes

### Technical Approach
1. Create `src/app/` directory
2. Move `App` struct definition to `app/mod.rs`
3. Move the `ApplicationHandler` impl block to `app/handler.rs`
4. Extract initialization functions (theme loading, config setup) to `app/initialization.rs`
5. Move `handle_menu_action()` and its helpers to `app/menu_actions.rs`
6. Reduce `main.rs` to just `fn main()` + event loop setup + `mod app`
7. Use `pub(crate)` for fields that need cross-module access within the app module

### Dependencies
Recommended to do CRT-T-0181 (theme switching helper) first to avoid double-moving code, but not strictly required.

### Risk Considerations
- **App struct field access:** The `ApplicationHandler` impl in `handler.rs` needs access to all App fields. Using `pub(crate)` on fields keeps them accessible within the crate while preventing external access.
- **Circular references:** The handler, menu actions, and initialization all reference App. Keep App definition in `mod.rs` and use `impl App` blocks in each sub-module.
