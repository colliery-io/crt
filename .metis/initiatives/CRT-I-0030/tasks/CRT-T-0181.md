# Extract theme switching helper, override getter macro, and tab dispatch cleanup

## Parent Initiative

[[CRT-I-0030]] — Readability and Module Decomposition

## Objective

Eliminate three patterns of code duplication: the 24-line theme switching sequence (duplicated in two places in main.rs), the 7 identical-pattern override getter methods in window.rs, and the 9 individual SelectTab match arms in main.rs.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] **Theme switching:** New `fn apply_theme_switch(&mut self, theme_name: &str)` method on App. Both call sites (menu action handler ~line 825 and context menu click ~line 1458) reduced to one-liners.
- [ ] **Override getters:** Macro or generic helper replaces 7 identical-pattern methods (`get_foreground`, `get_background`, `get_cursor_color`, `get_bold`, `get_italic`, `get_font_size_delta`, `get_opacity`) in `OverrideState`. Reduces ~56 lines to ~15.
- [ ] **Tab dispatch:** New `MenuAction::tab_index(&self) -> Option<usize>` method. 9 `SelectTab1..9` match arms replaced with `if let Some(idx) = action.tab_index()`.
- [ ] All existing tests pass unchanged
- [ ] Zero functional changes — behavior-preserving deduplication only

## Implementation Notes

### Technical Approach

**Theme switching helper:**
```rust
impl App {
    fn apply_theme_switch(&mut self, theme_name: &str) {
        // Load theme, update window state, reconfigure effects, etc.
        // Currently ~24 lines duplicated in two places
    }
}
```

**Override getter macro:**
```rust
macro_rules! override_getter {
    ($name:ident, $field:ident, $ty:ty) => {
        pub fn $name(&self) -> Option<$ty> {
            self.active.iter().filter(|o| o.is_active())
                .filter_map(|o| o.properties.$field).next_back()
        }
    };
}
override_getter!(get_foreground, foreground, Color);
override_getter!(get_background, background, Color);
// ... etc
```

**Tab dispatch:**
```rust
impl MenuAction {
    pub fn tab_index(&self) -> Option<usize> {
        match self {
            Self::SelectTab1 => Some(0),
            // ... through SelectTab9 => Some(8)
            _ => None,
        }
    }
}
```

### Dependencies
Best done before CRT-T-0179 (main.rs split) to avoid moving duplicated code that will then need deduplication.

### Risk Considerations
- **Override getter macro:** Macros can make code harder to navigate with IDE go-to-definition. The trade-off is worth it for 7 identical methods, but document the macro clearly.
