# Introduce EffectId enum and named constants

## Parent Initiative

[[CRT-I-0030]] — Readability and Module Decomposition

## Objective

Replace string-typed effect identifiers (magic strings like `"starfield"`, `"matrix"`) with a compile-time-safe `EffectId` enum, and replace magic numeric literals (e.g., `1.0 / 60.0`) with named constants.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] New `pub enum EffectId { Starfield, Particles, Matrix, Rain, Grid, Shape, Sprite }` defined (likely in `src/window/types.rs` or renderer effects module)
- [ ] `HashSet<String>` for tracking patched effects replaced with `HashSet<EffectId>` in `OverrideState`
- [ ] All `is_patched("starfield")` / `set_patched("starfield")` style calls converted to `is_patched(EffectId::Starfield)` etc.
- [ ] `EffectId` derives `Debug, Clone, Copy, PartialEq, Eq, Hash`
- [ ] `const ASSUMED_DT: f32 = 1.0 / 60.0;` (or similar) replaces the magic `1.0 / 60.0` literal in the render loop
- [ ] Any other magic numbers discovered during implementation replaced with named constants
- [ ] All existing tests pass unchanged (or updated to use EffectId enum)

## Implementation Notes

### Technical Approach
1. Define `EffectId` enum with variants for each known effect
2. Update `OverrideState` to use `HashSet<EffectId>` instead of `HashSet<String>`
3. Update `is_patched()`, `set_patched()`, `clear_patched()` signatures to take `EffectId`
4. Update all call sites — search for string literals matching effect names
5. Add `const ASSUMED_DT: f32 = 1.0 / 60.0;` near the render loop
6. Scan for other magic numbers and extract named constants

### Dependencies
None — can be done independently, but coordinates well with CRT-T-0178 (window module split) if EffectId lands in the window types module.

### Risk Considerations
- **Exhaustive matching:** Adding a new effect in the future requires updating the enum. This is actually a benefit — the compiler catches missing cases.
- **Existing tests:** Tests in `render/mod.rs` that use string-based `is_patched`/`set_patched` will need updating to use `EffectId` variants.
