# Split window.rs into window/ module directory

## Parent Initiative

[[CRT-I-0030]] — Readability and Module Decomposition

## Objective

Break `src/window.rs` (1,620 lines, 23+ public types) into a `src/window/` module directory with focused sub-modules, improving navigability and reducing merge conflict surface area.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `src/window.rs` replaced by `src/window/mod.rs` + sub-modules
- [ ] Target structure: `mod.rs` (WindowState + core lifecycle), `state.rs` (RenderState, CursorInfo, TextDecoration), `ui.rs` (UiState, ToastType, CopyIndicator, ZoomIndicator, BellState), `interaction.rs` (InteractionState, SelectionState, UrlDetection), `overrides.rs` (OverrideState, ActiveOverride, getters), `types.rs` (TabId, small enums, type aliases)
- [ ] All public types re-exported through `mod.rs` so existing `use crate::window::*` imports continue to work unchanged
- [ ] Zero logic changes — pure file reorganization
- [ ] All 229 existing tests pass without modification
- [ ] `cargo build --all-targets` succeeds on both macOS and Linux

## Implementation Notes

### Technical Approach
1. Create `src/window/` directory
2. Move types into sub-modules by grouping (state, ui, interaction, overrides, types)
3. Add `pub use` re-exports in `mod.rs` for all public types
4. Keep `WindowState` struct definition and core lifecycle methods (new, resize, close) in `mod.rs`
5. Move `#[cfg(test)] mod tests` block into the appropriate sub-module or keep in `mod.rs`

### Dependencies
None — this task is independent of all other CRT-I-0030 tasks.

### Risk Considerations
- **Import breakage:** Mitigated by re-exporting everything through `mod.rs`. Existing `use crate::window::*` and `use crate::window::SomeType` both continue to work.
- **Cross-module visibility:** Some fields may need `pub(crate)` if they were previously accessible within the single file. Watch for compilation errors.
