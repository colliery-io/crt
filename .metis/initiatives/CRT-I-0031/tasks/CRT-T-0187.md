# Partial grid rendering via damage tracking bounds

## Parent Initiative

[[CRT-I-0031]] — Performance and Memory Optimization

## Objective

Use alacritty_terminal's `TermDamage` line-level information to only rebuild cell data for changed lines, keeping previous frame data for undamaged lines. Currently the entire visible grid is re-rendered even when a single character is typed. Estimated 10-30% GPU work reduction for interactive use.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] Expose damage bounds (which specific lines changed) from `crt-core` Terminal, not just the boolean `has_damage()`
- [ ] Grid rendering path retains previous frame's cell data in a buffer
- [ ] Only damaged lines are rebuilt; undamaged lines reuse retained data
- [ ] Partial buffer updates submitted to GPU (only changed regions)
- [ ] Content hash still used for full-skip when nothing changed
- [ ] Frame time measurement shows improvement during interactive typing
- [ ] Full-screen scrolling still works correctly (all lines damaged = full rebuild)
- [ ] All existing tests pass

## Implementation Notes

### Technical Approach
1. In `crt-core`, expose `TermDamage` line ranges through a new method (e.g., `damaged_lines() -> Option<Range<usize>>`)
2. In the grid rendering path, maintain a `Vec<CellData>` from the previous frame
3. On partial damage, only iterate and rebuild cells for damaged line ranges
4. Copy undamaged regions from the retained buffer
5. Submit the combined buffer to the GPU

### Dependencies
Coordinates with the render pipeline but does not depend on other CRT-I-0031 tasks.

### Risk Considerations
- **High effort:** This touches both crt-core (damage exposure) and the rendering path. Careful testing needed.
- **Correctness:** Must handle edge cases: scrollback changes, resize (invalidates all lines), selection overlay changes.
- **Diminishing returns during scrolling:** When all lines are damaged (full-screen output), this adds overhead without benefit. Need to detect and fall back to full rebuild.

## Status Updates

- Added `damaged_line_set()` method to `Terminal` in crt-core — returns `Option<Vec<usize>>` (None = full damage, Some = partial line indices)

  - **Full damage/first render**: prepares all cells and caches results per-line for future reuse
