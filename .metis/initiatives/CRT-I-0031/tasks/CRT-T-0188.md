# Glyph atlas capacity monitoring and overflow handling

## Parent Initiative

[[CRT-I-0031]] — Performance and Memory Optimization

## Objective

Add capacity monitoring to the glyph atlas (`glyph_cache.rs`) and implement graceful handling when the 1024x1024 atlas fills up. Currently `allocate()` returns `None` silently, causing glyphs to fail to render with no warning or recovery.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] Log warning at 80% atlas capacity with stats (unique glyphs cached, utilization percentage)
- [ ] When atlas is full, implement one of: LRU eviction (clear least-recently-used glyphs and re-pack) or dynamic atlas growth (1024 → 2048)
- [ ] Decision between LRU vs growth documented based on trade-offs discovered during implementation
- [ ] Test with Unicode-heavy content (CJK + emoji mix) to verify graceful handling at capacity
- [ ] Existing glyph rendering continues to work correctly for normal ASCII content
- [ ] Atlas stats available at debug log level for profiling

## Implementation Notes

### Technical Approach
1. Add usage tracking to the glyph allocator (count of allocated regions, total area used)
2. Add capacity check after each allocation — log warning when crossing 80% threshold
3. **LRU eviction approach:** Track last-use timestamp per glyph entry. When full, evict oldest 25% and defragment. Simpler but causes momentary re-rendering stutter.
4. **Growth approach:** When full, create a 2048x2048 atlas, copy existing content, continue allocating. Uses 4x memory but avoids eviction stutter.
5. Recommend starting with growth (simpler, one-time cost) and adding eviction later if memory is a concern.

### Dependencies
None — self-contained in `crates/crt-renderer/src/glyph_cache.rs`.

### Risk Considerations
- **Atlas growth:** Requires creating a new GPU texture and copying. Must happen between frames to avoid artifacts.
- **LRU eviction:** Re-rendering evicted glyphs on next use causes a brief stutter. Acceptable for rare event.

## Status Updates

### Completed:
- Added capacity monitoring to `AtlasPacker`: `pixels_allocated` counter, `utilization()` method
- Logs warning at 80% atlas capacity with stats (glyph count, utilization, atlas dimensions)
- **Overflow handling**: When atlas is full, clears all cached glyphs and resets packer. Actively visible glyphs re-cache on next frame (effective LRU-by-visibility eviction). Chose clear-and-rebuild over atlas growth because it avoids needing `device` reference at allocation time.
- Added `atlas_stats()` public method returning (glyph_count, utilization, width, height) for debug logging
- Added `reset()` method to `AtlasPacker`, used by both overflow recovery and `set_font_size()`
- Build clean, all 46 tests pass
