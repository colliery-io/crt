# Replace Mutex event queue with lock-free crossbeam SegQueue

## Parent Initiative

[[CRT-I-0031]] — Performance and Memory Optimization

## Objective

Replace the `Arc<Mutex<EventStorage>>` in `TerminalEventProxy` (`crates/crt-core/src/lib.rs:70-115`) with a lock-free `crossbeam::queue::SegQueue`. While events are infrequent, `take_events()` is called every frame, and the Mutex adds unnecessary latency on the rendering hot path.

## Implementation Notes

### Technical Approach
```rust
use crossbeam::queue::SegQueue;

pub struct TerminalEventProxy {
    events: Arc<SegQueue<Event>>,
}

impl EventListener for TerminalEventProxy {
    fn send_event(&self, event: Event) {
        self.events.push(event);
    }
}

impl TerminalEventProxy {
    pub fn take_events(&self) -> Vec<Event> {
        let mut events = Vec::new();
        while let Some(event) = self.events.pop() {
            events.push(event);
        }
        events
    }
}
```

### Dependencies
Adds `crossbeam` crate dependency. This is a widely-used, well-maintained crate in the Rust ecosystem with minimal overhead.

### Risk Considerations
- **Low risk:** The replacement is a direct API swap. `SegQueue` is MPSC-safe and used extensively in production Rust systems.
- **Memory ordering:** `SegQueue` handles all ordering internally — no manual atomic operations needed.
