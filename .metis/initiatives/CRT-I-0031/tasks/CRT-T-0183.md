# Fix string double-allocation in all_lines_text()

## Parent Initiative

[[CRT-I-0031]] — Performance and Memory Optimization

## Objective

Eliminate the double-allocation in `all_lines_text()` (`crates/crt-core/src/lib.rs:481-503`) where `collect()` creates a String and then `trim_end().to_string()` creates a second copy. Use in-place `truncate()` instead. Estimated 10-20% improvement for search/log operations that call this hot path.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `all_lines_text()` builds each line string with `String::with_capacity(columns)` and uses `truncate()` for in-place trimming instead of `trim_end().to_string()`
- [ ] Pre-allocate the result `Vec` with `Vec::with_capacity(total_lines)`
- [ ] Also fix debug logging per-byte allocation in `lib.rs:562-586` if present (batch hex formatting)
- [ ] All existing tests pass unchanged
- [ ] Micro-benchmark shows reduced allocation count per call

## Implementation Notes

### Technical Approach
```rust
pub fn all_lines_text(&self) -> Vec<(i32, String)> {
    let mut lines = Vec::with_capacity(total_lines);
    for row in grid_rows {
        let mut text = String::with_capacity(self.columns());
        for cell in row { text.push(cell.c); }
        let trimmed_len = text.trim_end().len();
        text.truncate(trimmed_len);  // In-place, no second allocation
        lines.push((line_idx, text));
    }
    lines
}
```
