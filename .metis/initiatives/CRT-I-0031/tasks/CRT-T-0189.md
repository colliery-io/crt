# Criterion benchmark suite with CI regression detection

## Parent Initiative

[[CRT-I-0031]] — Performance and Memory Optimization

## Objective

Create a criterion benchmark suite for CRT's hot paths and integrate it into CI with regression detection. Currently `src/bin/benchmark.rs` exists for manual end-to-end profiling but there's no automated performance regression detection.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] Criterion benchmarks added for: terminal text processing throughput (bytes/second), glyph cache lookup, content hash computation, theme CSS parsing
- [ ] Benchmarks organized in `benches/` directory following criterion conventions
- [ ] CI workflow step runs benchmarks and compares against baseline (criterion's built-in comparison)
- [ ] Performance regression >10% triggers a CI warning (not failure — benchmarks can be noisy)
- [ ] Existing `src/bin/benchmark.rs` retained for manual end-to-end profiling
- [ ] `cargo bench` works locally for developer use

## Implementation Notes

### Technical Approach
1. Create `benches/` directory with criterion benchmark groups
2. **Text processing:** Benchmark `all_lines_text()` and `process_input()` with various buffer sizes (1KB, 10KB, 100KB)
3. **Glyph cache:** Benchmark lookup hit/miss for ASCII and Unicode character sets
4. **Content hash:** Benchmark hash computation on typical 80x24 and 200x50 terminal grids
5. **Theme parsing:** Benchmark CSS theme loading for small/medium/large theme files
6. Add CI workflow step: `cargo bench -- --output-format bencher` with comparison
7. Store baseline results as CI artifact for cross-run comparison

### Dependencies
`criterion` is already in `dev-dependencies`. No new dependencies needed.

### Risk Considerations
- **CI timing noise:** Shared CI runners have variable performance. Use criterion's statistical analysis (confidence intervals) and set regression threshold at 10% to avoid false positives.
- **Benchmark stability:** Terminal processing benchmarks may need deterministic input (pre-generated escape sequences) to avoid variability from random data.
