# PTY read buffer reuse to eliminate per-read Vec allocations

## Parent Initiative

[[CRT-I-0031]] — Performance and Memory Optimization

## Objective

Eliminate per-read `Vec<u8>` allocations in PTY reader thread (`pty.rs:147-164`) by reusing buffers. Currently `buf[..n].to_vec()` allocates a new Vec on every read, which at typical shell output rates means hundreds of allocator calls per frame during scrolling.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] PTY reader thread reuses buffers instead of calling `.to_vec()` on every read
- [ ] Implementation uses either a buffer pool (checkout/return pattern) or a ring buffer approach
- [ ] Channel still delivers owned data to the consumer (may use `Arc<[u8]>` or pooled `Vec<u8>`)
- [ ] `PtyBackend` trait and `Pty` public API unchanged
- [ ] All PTY unit tests and shell integration tests pass unchanged
- [ ] Benchmark with `scrolling_output` scenario shows measurable reduction in allocations

## Implementation Notes

### Technical Approach
Option A — Buffer pool:
```rust
struct BufferPool {
    buffers: Vec<Vec<u8>>,
}
impl BufferPool {
    fn checkout(&mut self) -> Vec<u8> {
        self.buffers.pop().unwrap_or_else(|| Vec::with_capacity(4096))
    }
    fn return_buf(&mut self, mut buf: Vec<u8>) {
        buf.clear();
        if self.buffers.len() < 16 { self.buffers.push(buf); }
    }
}
```

Option B — Ring buffer with `Arc<[u8]>` slices to avoid copies entirely. Needs benchmarking to compare.

### Dependencies
None — self-contained in `crates/crt-core/src/pty.rs`. Note: CRT-T-0180 (PTY spawn consolidation from CRT-I-0030) touches the same file but different code paths.

### Risk Considerations
- Buffer pool adds complexity to the reader thread. Keep it simple — a local pool in the thread closure is sufficient, no need for cross-thread sharing.
- Must not change the semantics of the output channel — consumer expects owned data it can process at its own pace.

## Status Updates

- Added recycle channel (`recycle_tx`/`recycle_rx`) to `Pty` struct for buffer reuse between consumer and reader thread

- `read_available()` sends consumed buffers back via `recycle_tx` after `extend_from_slice`
- After steady state, reader thread reuses previously allocated buffers instead of allocating per read
