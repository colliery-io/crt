## Implementation Notes

### Technical Approach

**Effect tests:** Configure CRT post-processing effects at known settings, render, compare:
```rust
#[test]
fn visual_crt_scanlines() {
    let renderer = HeadlessRenderer::new(640, 480).unwrap();
    let mut terminal = Terminal::new(Size::new(80, 24));
    terminal.process_input(b"Scanline test content\r\n");
    
    let theme = Theme::default();  // Has CRT effects enabled
    render_with_effects(&renderer, &terminal, &theme, CrtEffectSettings {
        scanline_intensity: 0.5,
        curvature: 0.0,
        ..Default::default()
    });
    
    let actual = renderer.capture_png();
    assert_visual_match!(actual, "crt_scanlines", tolerance = 1.0);
}
```

**CI workflow addition:**
```yaml
visual-tests:
  runs-on: ${{ matrix.os }}
  strategy:
    matrix:
      os: [macos-latest, ubuntu-latest]
  steps:
    - uses: actions/checkout@v4
    - name: Install deps (Linux)
      if: runner.os == 'Linux'
      run: sudo apt-get install -y mesa-utils libegl1-mesa-dev
    - name: Run visual tests
      run: cargo test --test visual_tests
    - name: Upload failure artifacts
      if: failure()
      uses: actions/upload-artifact@v4
      with:
        name: visual-test-failures-${{ matrix.os }}
        path: tests/visual/failures/
```

### Risk Considerations
- **Effect rendering determinism:** CRT effects use shaders that may produce slightly different results across GPU drivers. Use higher tolerance (1.0-2.0%) for effect tests vs text tests (0.5%).
- **CI GPU support:** Software rendering on CI may not support all shader features. Test early with a simple "does wgpu initialize" check and skip gracefully if not.
