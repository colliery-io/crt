# Build HeadlessRenderer with offscreen wgpu rendering and frame capture

## Parent Initiative

[[CRT-I-0032]] — Visual Regression Testing Infrastructure

## Objective

Build a `HeadlessRenderer` that creates a wgpu device without a window surface, renders terminal state to an offscreen texture, and captures the result as PNG bytes. This is the foundational infrastructure for all visual regression tests.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `HeadlessRenderer` struct in `crates/crt-renderer/src/headless.rs`
- [ ] Creates wgpu device with `force_fallback_adapter: true` for software rendering compatibility
- [ ] Renders to an offscreen texture with `COPY_SRC` usage
- [ ] `capture_frame() -> Vec<u8>` reads back RGBA pixels via staging buffer
- [ ] `capture_png() -> Vec<u8>` encodes captured frame as PNG
- [ ] Exposes `device()`, `queue()`, `texture_view()` for pipeline setup
- [ ] Works on both macOS (Metal) and Linux (Vulkan/GL software)

- [ ] Unit test: verify PNG encoding produces valid image with correct dimensions

## Implementation Notes

### Technical Approach
```rust
pub struct HeadlessRenderer {
    device: wgpu::Device,
    queue: wgpu::Queue,
    render_texture: wgpu::Texture,
    render_view: wgpu::TextureView,
    staging_buffer: wgpu::Buffer,
    width: u32,
    height: u32,
}

impl HeadlessRenderer {
    pub fn new(width: u32, height: u32) -> Result<Self, HeadlessError>;
    pub fn texture_view(&self) -> &wgpu::TextureView;
    pub fn capture_frame(&self) -> Vec<u8>;
    pub fn capture_png(&self) -> Vec<u8>;
    pub fn device(&self) -> &wgpu::Device;
    pub fn queue(&self) -> &wgpu::Queue;
}
```

Key implementation details:
1. `request_adapter` with `power_preference: LowPower` for CI
2. Render texture format: `Bgra8UnormSrgb` (matching the existing pipeline)
3. Staging buffer size: `width * height * 4` bytes with `MAP_READ | COPY_DST`
4. Frame capture: `encoder.copy_texture_to_buffer()` → `buffer.slice(..).map_async()` → read bytes
5. PNG encoding via the `image` crate (already a dependency)

### Dependencies
None — this can be built independently. Uses existing wgpu infrastructure patterns from `src/gpu/`.

### Risk Considerations
- **CI GPU availability:** GitHub Actions runners may not have GPU drivers. `force_fallback_adapter: true` enables CPU-based software rendering (wgpu's "llvmpipe" backend on Linux, software Metal on macOS). Must verify this works in CI.
- **Texture format matching:** The headless renderer's texture format must match what the existing rendering pipelines expect, or they won't be composable.
