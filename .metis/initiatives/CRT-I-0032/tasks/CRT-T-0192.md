## Implementation Notes

### Technical Approach
Each test follows this pattern:
```rust
#[test]
fn visual_text_basic_ascii() {
    let renderer = HeadlessRenderer::new(640, 480).unwrap();
    let mut terminal = Terminal::new(Size::new(80, 24));
    terminal.process_input(b"Hello, World!\r\n");
    
    // Render terminal state to headless renderer
    render_terminal_to_headless(&renderer, &terminal, &theme);
    
    let actual = renderer.capture_png();
    assert_visual_match!(actual, "text_basic_ascii");
}
```

A shared `render_terminal_to_headless()` helper wires the terminal's renderable content into the existing rendering pipeline targeting the headless renderer's texture.

### Dependencies
- CRT-T-0190 (HeadlessRenderer) — must be complete

### Risk Considerations
- **Font availability:** CI runners may have different fonts. Use a bundled test font or accept platform-specific golden files.
- **Terminal state setup:** Some tests need specific terminal states (cursor at position, selection active). Use the existing Terminal API for programmatic setup.
