# Golden file comparison framework with perceptual diff and platform awareness

## Parent Initiative

[[CRT-I-0032]] — Visual Regression Testing Infrastructure

## Objective

Implement a golden file comparison framework that compares captured frames against reference images with configurable perceptual diff tolerance, generates diff images on failure, and supports per-platform golden files (macOS vs Linux font rendering differences).

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `compare_with_golden(actual, golden_path, tolerance) -> ComparisonResult` function
- [ ] `ComparisonResult` includes: matched (bool), diff_percentage (f64), diff_pixels (usize), total_pixels (usize), diff_image (Option PNG bytes)
- [ ] Diff image highlights changed pixels in red for easy visual inspection
- [ ] Platform-aware golden file selection: `tests/visual/golden/{test_name}.{macos|linux}.png`
- [ ] `UPDATE_GOLDEN=1` environment variable causes tests to write actual as new golden file instead of comparing
- [ ] Failed tests save actual + diff images to `tests/visual/failures/` for inspection
- [ ] Default tolerance: 0.5% pixel difference allowed (configurable per test)
- [ ] Unit tests: identical images match, slightly different images within tolerance match, different images fail, diff image generated correctly

## Implementation Notes

### Technical Approach

```rust
    pub matched: bool,

// Helper macro for visual tests
macro_rules! assert_visual_match {
    ($actual:expr, $name:expr) => { ... };
    ($actual:expr, $name:expr, tolerance = $tol:expr) => { ... };

```

Directory structure:
```
tests/visual/
  golden/           # Reference images (checked into git)
    text_basic.macos.png
    text_basic.linux.png
    cursor_block.macos.png
    ...
  failures/         # Generated on test failure (gitignored)
    text_basic.actual.png
    text_basic.diff.png
  compare.rs        # Comparison framework
```

Platform detection: `cfg!(target_os = "macos")` → ".macos.png", else → ".linux.png"

### Dependencies
Depends on CRT-T-0190 (HeadlessRenderer) for generating actual frames.

### Risk Considerations
- **Anti-aliasing variance:** Even on the same platform, font rendering may differ by OS version. Keep tolerance configurable and document what versions golden files were generated on.
- **Git storage:** PNG golden files add to repo size. Keep them small (test at modest resolutions like 640x480) and use `git lfs` if they grow large.
