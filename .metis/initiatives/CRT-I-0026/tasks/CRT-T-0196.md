# Tab reordering within a window

## Objective

Implement tab reordering via drag within the same window's tab bar. This is the simplest drag operation — no cross-window concerns — and validates the drag infrastructure end-to-end.

### User interaction
1. User presses on a tab and drags horizontally
2. An insertion caret appears between tabs showing where the tab will land
3. User releases → tab moves to the new position
4. Active tab selection is preserved (the moved tab stays selected)

## Implementation Notes

### TabBarState changes (crates/crt-renderer/src/tab_bar/state.rs)

```rust
/// Move a tab from one index to another
pub fn move_tab(&mut self, from: usize, to: usize) {
    if from == to || from >= self.tabs.len() || to >= self.tabs.len() {
        return;
    }
    let tab = self.tabs.remove(from);
    self.tabs.insert(to, tab);
    // Update active_tab index to follow the moved tab
    // ...
}
```

The `active_tab` index tracking needs care: if the active tab is the one being moved, update to `to`. If the active tab is between `from` and `to`, it shifts by ±1.

### Insertion index computation (src/input/drag.rs)

Given cursor x-position and the tab rects from `TabLayout`, find the insertion gap:
- For each tab rect, if cursor is in the left half → insert before, right half → insert after
- This is a pure function: `compute_reorder_index(cursor_x: f32, tab_rects: &[TabRect], dragged_index: usize) -> usize`
