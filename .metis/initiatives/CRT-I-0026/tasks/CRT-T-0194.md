# Global tab IDs and drag state types

## Objective

Lay the foundation for tab drag-and-drop by introducing globally unique tab IDs (fixing a collision problem) and the core drag state types that all subsequent tasks depend on.

### Problem: Tab ID Collisions

Currently `TabBarState` uses a per-window `next_id: u64` counter starting at 0. Every window's first tab is id=0, second is id=1, etc. When we move a tab between windows, the moved tab's ID can collide with an existing tab in the target window. This must be fixed before any cross-window tab operations.

**Solution**: Move the ID counter to `App` (or use a global `AtomicU64`). All tab creation goes through a single counter so IDs are unique across all windows.

### New types

```rust
// src/input/drag.rs
#[derive(Debug, Clone, PartialEq)]
pub enum DragDropTarget {
    Reorder { insert_index: usize },
    Merge { target_window_id: WindowId, insert_index: usize },
    Detach,
    Pending,
}

pub struct TabDragState {
    pub tab_id: TabId,
    pub source_window_id: WindowId,
    pub start_pos: PhysicalPosition<f64>,
    pub current_pos: PhysicalPosition<f64>,
    pub drop_target: DragDropTarget,
    pub drag_active: bool,
}
```

### Global ID approach
Change `TabBarState::add_tab` signature from `add_tab(&mut self, title) -> u64` to `add_tab(&mut self, id: u64, title)` (caller provides ID). Add `App::next_tab_id()` that increments and returns.
