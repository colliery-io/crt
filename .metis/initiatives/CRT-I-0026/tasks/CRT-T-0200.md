# Drag visual feedback

## Objective

Add visual feedback during tab dragging: a ghost tab at the cursor, insertion carets showing where the tab will land, and a detach indicator when outside all windows. This makes the drag feel responsive and predictable.

### Visual elements
1. **Ghost tab** — semi-transparent copy of the dragged tab rendered at the cursor position
2. **Insertion caret** — a vertical line or highlight between tabs showing the drop position (for reorder and merge)
3. **Detach indicator** — a visual cue on the ghost tab when in detach mode (e.g., a "+" badge or the ghost detaching/floating)
4. **Source tab dimming** — the original tab in the source bar is dimmed/grayed during the drag

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] A ghost tab follows the cursor during active drag (after threshold)
- [ ] Ghost tab is semi-transparent (alpha ~0.7) and shows the tab's title
- [ ] An insertion caret (vertical line) appears at the target index during Reorder
- [ ] An insertion caret appears on the target window's tab bar during Merge
- [ ] A distinct visual indicator shows when in Detach mode
- [ ] The source tab is visually dimmed during the drag
- [ ] All visual feedback disappears immediately on drop or cancel
- [ ] Performance: drag rendering doesn't cause frame drops (ghost rendered in existing render pass)

## Implementation Notes

### Ghost tab rendering

The ghost tab needs to render on top of everything, following the cursor across windows. Options:

**Option A: Render in the tab bar pass with offset.** During the render pass, if a drag is active, render an extra tab at the cursor position with reduced alpha. Simple but only works within the source window.

**Option B: OS-level drag image.** macOS supports `NSWindow` drag images via `beginDraggingSession`. This gives a true floating image that works across windows. More platform-specific.

**Option C: Overlay window.** Create a small, transparent, always-on-top utility window that follows the cursor. Cross-platform via winit but adds complexity.

**Recommended**: Start with Option A (in-window rendering) for the initial implementation. It covers reorder perfectly and gives approximate feedback for detach/merge. Option B or C can be a follow-up for polish.

### Insertion caret

During tab bar rendering (in `TabBar::render_shapes_to_rects` or the vello path), check if a drag is active and if the drop target is Reorder/Merge. If so, draw a 2px-wide vertical line at the insertion gap position using the active tab accent color.

The insertion index comes from `TabDragState.drop_target`. The x-position is computed from `TabLayout::tab_rects()`.

### Source tab dimming

When rendering tabs, if a tab's ID matches the dragged tab, render it with reduced alpha.

### Passing drag state to renderer

The tab bar renderer currently doesn't know about drag state. Options:
- Pass `Option<&TabDragState>` to `TabBar::prepare()` / render methods
- Add a `drag_feedback: Option<DragFeedback>` field to `TabBar` that's set before rendering

**Recommended**: Add a lightweight `DragFeedback` struct to `TabBar`:
```rust
pub struct DragFeedback {
    pub dragged_tab_id: u64,
    pub insertion_index: Option<usize>,
    pub ghost_position: Option<(f32, f32)>,  // cursor position for ghost
    pub mode: DragMode,  // Reorder, Merge, Detach
}
```
