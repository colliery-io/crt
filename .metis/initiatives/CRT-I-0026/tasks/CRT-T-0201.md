# Edge cases, multi-monitor, and test suite

## Objective

Handle edge cases, multi-monitor scenarios, and add a test suite for the drag state machine. This is the hardening task that makes the feature production-ready.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] Escape key cancels drag at any point, restoring original tab position/state
- [ ] Multi-monitor: drag works when cursor crosses monitor boundaries
- [ ] Multi-monitor: new window from detach appears on the correct monitor (the one the cursor is on)
- [ ] Window focus: after detach, the new window is focused; after merge, the target window is focused
- [ ] Rapid operations: opening a new tab immediately after a drag doesn't corrupt state
- [ ] Tab editing: a tab in rename mode cannot be dragged (edit state takes priority)
- [ ] Context menu: drag is not initiated while context menu is open
- [ ] Unit tests for `TabDragState` transitions (Pending → Reorder/Merge/Detach → executed)
- [ ] Unit tests for `resolve_drop_target` with multi-window scenarios
- [ ] Unit tests for `move_tab`, `remove_tab`, `insert_existing_tab` edge cases
- [ ] Integration-level test: extract tab from one TabBarState, insert into another, verify both states are consistent

## Implementation Notes

### Multi-monitor

winit's `Window::current_monitor()` and `MonitorHandle::position()` / `size()` provide the info needed. When detaching, place the new window on whichever monitor contains the cursor. `Window::set_outer_position()` works in screen coordinates that span monitors.

### Escape key handling

In `src/input/keyboard.rs`, add a check: if Escape is pressed and `App.drag_state.is_some()`, cancel the drag (set `drag_state = None`) and return early (don't pass Escape to the terminal).

### Guard conditions

Before initiating a drag, check:
- `tab_bar.is_editing()` → skip drag, let edit mode handle the click
- `context_menu.visible` → skip drag, let context menu handle the click
- `tab_count <= 1` → skip drag (last-tab guard)

### Test organization

Add a `tests/drag_tests.rs` or `src/input/drag.rs` `#[cfg(test)]` module with:
- State machine transition tests (pure, no GPU)
- `resolve_drop_target` with mock window rects
- `move_tab` / `remove_tab` / `insert_existing_tab` with various tab configurations
- Round-trip: extract from A, insert into B, verify A and B state

### Files to modify
- `src/input/keyboard.rs` — Escape cancel
- `src/input/drag.rs` — guard condition helpers, tests
- `crates/crt-renderer/src/tab_bar/state.rs` — additional tests
- Possibly `tests/drag_tests.rs` for integration tests
