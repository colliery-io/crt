# Config and theme registry edge case tests

## Parent Initiative

[[CRT-I-0029]]

## Objective

Add edge case tests for configuration parsing and the theme registry. Config has 31 tests but gaps remain around keybinding validation and error recovery. The theme registry (`src/theme_registry.rs`) has zero tests despite being critical for theme switching and hot-reload.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] 10-15 new tests added
- [ ] Theme registry: loading from disk, cache behavior, theme switching, missing/invalid theme file handling
- [ ] Config: keybinding conflict detection, invalid key names, modifier parsing edge cases
- [ ] Config merge: user config overrides defaults correctly, partial configs don't lose unset fields
- [ ] All existing 31 config tests still pass

## Implementation Notes

### Files to test
- `src/config.rs` — Configuration parsing and merging (31 existing tests)
- `src/theme_registry.rs` — Theme loading, caching, switching (0 tests)

### Test areas
1. **Theme registry**: Load built-in theme, load from file path, cache hit on second load, switch theme, handle missing file gracefully, handle malformed CSS gracefully
2. **Config keybindings**: Parse `Cmd+Shift+T`, invalid modifier name, duplicate binding detection, empty keybinding list
3. **Config merging**: User specifies only `font_size` — verify all other fields use defaults, user overrides nested config (e.g., `effects.starfield.speed`)
4. **Error recovery**: Malformed TOML/YAML in config file, missing required fields, unknown fields ignored

### Approach
- Theme registry likely requires filesystem setup (temp dir with theme files) — use `tempfile` crate
- Config tests can use string parsing without filesystem
- Check existing test patterns in `config.rs` for consistency
