# CI coverage tracking with cargo-llvm-cov

## Parent Initiative

[[CRT-I-0029]]

## Objective

Add code coverage tracking to the CI pipeline using `cargo-llvm-cov`. The goal is to track and report coverage, not gate on a percentage. This gives visibility into which areas benefit most from additional tests.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] New GitHub Actions workflow (or step in existing `test.yml`) runs `cargo-llvm-cov`
- [ ] Coverage report generated in LCOV or HTML format
- [ ] Coverage summary printed in CI logs (line/function percentages)
- [ ] Coverage report uploaded as CI artifact for download
- [ ] Does NOT gate PRs on coverage percentage — informational only
- [ ] Works on both macOS and Ubuntu CI matrix

## Implementation Notes

### Approach
1. **Add coverage step** to `.github/workflows/test.yml` (or create separate `coverage.yml`)
2. **Install `cargo-llvm-cov`** via `cargo install` or GitHub Action (`taiki-e/install-action`)
3. **Run coverage**: `cargo llvm-cov --all-targets --lcov --output-path lcov.info`
4. **Upload artifact**: Use `actions/upload-artifact` to save the LCOV report
5. **Print summary**: `cargo llvm-cov --all-targets` (text output) for CI log visibility

### Considerations
- `cargo-llvm-cov` requires `llvm-tools-preview` rustup component — add to CI setup
- Coverage on macOS vs Linux may differ due to platform-specific code (`#[cfg(target_os)]`)
- Consider running coverage only on one platform (Linux) to save CI minutes, with regular tests on both
- HTML report is nice for local use but large for CI artifacts — LCOV is compact and parseable

### Optional future enhancements (not in scope)
- Codecov/Coveralls integration for PR comments
- Coverage diff on PRs (requires baseline)
- Coverage badges in README
