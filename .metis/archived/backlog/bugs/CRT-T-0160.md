## Implementation Notes

### Root Cause Found

The debug logging added for paste investigation used unsafe byte slicing:
```rust
&content[..content.len().min(50)]
```

This panics when byte 50 falls in the middle of a multi-byte UTF-8 character (emoji, non-ASCII text), because Rust string slicing requires char boundaries.

### Fix Applied

Changed to safe character-based slicing:
```rust
let preview: String = content.chars().take(50).collect();
```

### Related Code
- `src/input/mod.rs:895`: Paste content preview logging (fixed)
