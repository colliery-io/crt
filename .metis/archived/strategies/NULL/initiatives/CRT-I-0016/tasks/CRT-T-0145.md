# Move bell flash to CSS theming

## Objective

Make bell visual effects fully theme-driven via CSS `::on-bell` instead of hardcoded in `config.toml`.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] Add `--flash-color` property to `::on-bell` (optional, omit = no flash)
- [ ] Add `--flash-intensity` property to `::on-bell` 
- [ ] Use theme's `--duration` for flash duration
- [ ] `render_bell_flash()` reads from active override instead of config
- [ ] No flash properties in CSS = no screen flash (sprite effects only)
- [ ] Deprecate `[bell] visual`, `flash_duration_ms`, `flash_intensity` from config.toml
- [ ] Keep `[bell] audible` for system sound (if implemented)

## Implementation Notes

### New CSS Properties

```css
:terminal::on-bell {
    --duration: 300ms;
    
    /* Screen flash (optional) */
    --flash-color: #ffff00;
    --flash-intensity: 0.5;
    
    /* Sprite/other effects */
    --sprite-fps: 24;
}
```

### Changes Required

1. **crt-theme**: Add `flash_color: Option<Color>` and `flash_intensity: Option<f32>` to `EventOverride`
2. **parser.rs**: Parse `--flash-color` and `--flash-intensity` in `apply_event_properties()`
3. **render.rs**: Check `state.ui.overrides.get_flash_color()` in `render_bell_flash()`
4. **config.rs**: Mark bell visual config as deprecated
